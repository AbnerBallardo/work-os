# Technology Committee - Topic Brief Template

Use this template only after a topic passes triage and is accepted for committee review.

Possible topics should start with [Technology Committee - Triage Submission Template.md](Technology%20Committee%20-%20Triage%20Submission%20Template.md).

Incomplete full briefs should be rejected, redirected, or converted to async pre-read.

---

## Output Rules

Default format: Word Paste Mode.

Use this format when content will be copied from M365 Copilot into Word.

Rules:

* Target one page.
* Maximum 900 words unless the chair requests more detail.
* Use plain labels and short lines.
* Do not use Markdown tables, pipe tables, code fences, HTML, bold, italics, or decorative separators.
* Limit alternatives to two, trade-offs to three, and risks / constraints to three.
* Include only decision-relevant context.
* Put gaps under `Assumptions / missing inputs` instead of expanding narrative.
