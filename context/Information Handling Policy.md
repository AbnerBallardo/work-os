## Classification

**Repository classification:** Highly Sensitive

This repository contains:

* Private Work operating context
* Political and leadership context
* Work governance and committee rules
* Agent runtime behavior and source-boundary packages
* Compressed Work memory and operational case patterns

Default rule:

* Treat all repository content as **Highly Sensitive** unless explicitly prepared for a controlled runtime, corporate, or external use.

---

## Handling Rules

### Storage

* Store only in trusted private repositories or controlled private folders.
* Do not mirror to public repositories.
* Do not use Work OS as a replacement for corporate systems of record.
* Do not paste raw Work OS context into third-party tools unless the specific content is required and the exposure is acceptable.
* Treat generated runtime packages as upload artifacts, not long-term storage.

---

### Content Admission

Do not store:

* Credentials, tokens, or secrets
* Raw HR, legal, audit, regulatory, or financial records
* Screenshots, exports, or attachments copied directly from corporate systems
* Raw meeting logs, incident chatter, or short-lived delivery noise
* Name-level workforce action lists
* Unreviewed political notes in Work-facing runtime packages

Store only:

* Durable Work operating context
* Reusable governance and committee rules
* Compressed Work memory and validated observations
* Stable agent behavior and source-boundary rules
* Sanitized runtime package sources required for controlled Work support

---

### Externalization

Before sharing or packaging any Work OS content externally:

1. Confirm the target surface: private Work OS reasoning, ChatGPT Project, M365 agent, OneDrive source package, corporate system, or public output.
2. Preserve sensitive detail only inside private Work OS reasoning when it materially improves judgment.
3. Generalize political, leadership, stakeholder, or institutional detail before use in Work-facing agents or public/content outputs.
4. Include only the minimum source material needed for the runtime role.
5. Treat copied text as potentially leaving the controlled environment.

---

### AI Usage

* Do not treat AI tools as private by default.
* Apply minimum necessary exposure.
* Work OS-owned private ChatGPT Project packages may include selected Work OS rules needed for recurring private reasoning and source-boundary review.
* Work-facing M365, Copilot, OneDrive, or institutional agent packages must use sanitized source packages only.
* Do not connect raw political notes, leadership assessments, broad Work context, memory, or unreviewed working notes to Work-facing agents.
* Generalize sensitive Work context into safe behavior, communication, routing, or review rules before external use.

---

## Operating Rule

Work OS is a private Work operating system and controlled Work support layer.

It may preserve sensitive Work context when that context improves private judgment, governance quality, or source-boundary decisions.

It must not become a raw mirror of corporate systems or a broad source package for Work-facing agents.

Use this decision rule:

| Context | Exposure rule |
|---|---|
| Private Work OS source files | Preserve stable sensitive context when it materially improves Work judgment. |
| Work OS-owned private ChatGPT Project packages | Include only selected source excerpts needed for recurring Work OS reasoning and maintenance. |
| Work-facing M365, Copilot, or OneDrive agent packages | Use sanitized runtime sources only; exclude raw political, leadership, memory, and broad context files. |
| Corporate systems | Use official records and approved channels; do not treat Work OS as the corporate source of record. |
| Personal OS | Share only operator-level Work overview and cross-domain constraints. |
| Content OS or public outputs | Generalize Work details into reusable frameworks without exposing confidential context. |
| Temporary operational material | Do not promote unless it becomes durable Work governance, memory, or source-boundary logic. |

If a piece of information is useful but external exposure would create asymmetric downside:

* Keep the private decision context in Work OS.
* Generalize or redact the detail in the runtime or externalized artifact.
